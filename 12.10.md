### Redis事务
#### 语法
| 指令  |  作用 | 特点 |
|---|---|---|
| `multi`  |  开始事务 | |
| `exec`  |  执行事务中的命令 | 不会被打断,遇到错误继续执行,不能回滚 |
| `discard`  |  放弃事务 | |
| `watch`  | 结合`exec`实现乐观锁  | |
#### 简单使用
```
> set n 1
ok
> multi
ok
> set n 2
QUEUED
> incr n
QUEUED
> exec
ok
(integer 3)
> get n
"3"
```
#### 特性
> 隔离性: 不会被其他客户端发送来的命令请求所打断

> 原子性: 事务中的命令要么全部被执行，要么全部都不执行

> 无回滚：实现简单
